# Task 30: Full Frontend Redesign

## Context

Current frontend: Next.js 15 + React 19, vanilla CSS tokens in globals.css (650 lines), 10 UI components, 12 pages, framer-motion for animations. Existing design is functional but generic — slate/monochrome palette, standard card-based layouts. User explicitly wants a **distinctive, purpose-built design** for a tutor-student product in history/social studies, not another SaaS template.

Key constraints from brief:
- Tutor: fast, informative tool (like good CRM), used many times daily
- Student: anxiety-reducing exam interface, no aggressive timers/red colors
- Must preserve ALL existing functionality and API contracts
- UI verification mandatory after every form change (consent/consent_152fz lesson)
- Adaptivity + accessibility (prefers-reduced-motion, keyboard focus)

---

## Part 1: Design Plan

### Palette: "Archivist's Desk"

Named for the product's subject domain — documents, sources, measured scholarship. NOT "aged paper + serifs" (generic), but the quiet authority of a well-organized workspace.

| Token | Hex | Role |
|-------|-----|------|
| `--c-ink` | `#1a1a2e` | Primary text, sidebar active, buttons. Deep navy-black, not pure black — warmer, less harsh |
| `--c-parchment` | `#f8f6f0` | Page background. Warm off-white, subtly suggests paper without being literal |
| `--c-surface` | `#ffffff` | Cards, modals, inputs. Clean white against warm background |
| `--c-accent` | `#c9a96e` | Accent gold. Used sparingly: active states, progress highlights, key CTAs. Evokes "important document" without being gaudy |
| `--c-success` | `#4a7c59` | Muted forest green for correct answers, completed states. Calming, not neon |
| `--c-danger` | `#b44a4a` | Muted brick red for errors. Warm, not alarming |
| `--c-muted` | `#8b8680` | Secondary text, borders, disabled states. Warm gray, not zinc |
| `--c-hover` | `#f0ece4` | Hover backgrounds. Warm tint, not cold gray |

**Why not pure black/white/zinc (current):** Current palette is indistinguishable from any Notion/Linear clone. The warm tones create an immediate "this is a different product" impression while remaining professional.

**Rejected alternatives:**
- Blue-primary (too corporate/edu-SaaS — Canvas, Blackboard)
- Green-primary (too "health/wellness")
- Dark mode primary (alienates older tutors, adds complexity)

### Typography

| Role | Stack | Size | Weight |
|------|-------|------|--------|
| Headings | `Inter, system-ui, sans-serif` | 18-28px | 600-700 |
| Body | `Inter, system-ui, sans-serif` | 14-15px | 400 |
| Data/Labels | `Inter, system-ui, sans-serif` | 12-13px | 500 |
| Task text (exam) | `Inter, system-ui, sans-serif` | 16px | 400 |
| Source quotes | `Inter, system-ui, sans-serif` | 15px | 400, italic |

**Why Inter (keep existing):** Already loaded via next/font with cyrillic subset. Excellent readability at small sizes (critical for dense tutor dashboards). Variable weight support. Switching font would add load time with no readability gain.

**Change from current:** Increase base from 14px to 15px. Current 14px is slightly cramped for long reading sessions. Task text bumped to 16px for exam readability.

### Layout Concept: "Dense Left, Calm Right"

**Tutor screens:** Two-panel layout with a dense, information-rich left sidebar/panel and a focused right content area. Think CRM: list of students on left, detail on right. High information density — the tutor scans many items quickly.

**Student screens:** Single-column, centered, generous whitespace. The test-taking screen should feel like a focused workspace — one question at a time, generous padding, large tap targets. Timer is subtle (pill in header, not a countdown clock).

**Signature element: "Source Card"** — A distinctive card variant used for essay/source-text tasks. It renders the inherited source text in a visually distinct container:
- Left border: 3px accent gold stripe
- Background: warm tint (`--c-hover`)
- Italic text for the quote body
- Small label "ИСТОЧНИК" (SOURCE) in uppercase muted text above

This element is unique to this product — no generic SaaS has a "source document card" as a first-class visual element. It immediately signals "this product understands academic work." It appears on:
- Test-taking screen (when task has inherited source text)
- Review screen (showing original source alongside student answer)
- Content preview (showing task with its source)

**Why this is NOT generic:** The source card solves a real product problem (students need to see the source text while answering) in a way that's visually distinctive and domain-specific. A generic template would just dump the text inline.

### What We're Keeping vs Changing

| Keep | Change |
|------|--------|
| All 10 UI components (refactor, not replace) | Color tokens, spacing, shadows |
| Sidebar navigation structure | Visual treatment (denser, warmer) |
| PageWrapper animation pattern | Motion timing (slower, calmer for student) |
| All API calls and data flow | Zero changes to API layer |
| Modal/Input/Select behavior | Visual polish only |
| Responsive breakpoint (768px) | Add student-specific breakpoints |

## Part 2: Two Key Screens (implement first, get approval)

### Screen 1: Test Creation (`tests/new/page.tsx`)

Current: Multi-step form with position toggles, subject selector, theme tree with FIPI counts.

**Changes:**
- Warmer card backgrounds, gold accent on active position toggles
- Denser layout: reduce vertical spacing, use 2-column grid for parameters
- Theme tree: show FIPI counts as small gold badges, not separate text
- "Create Test" button: accent gold, prominent placement
- EGE generator section: visually separated with a subtle divider

### Screen 2: Test Taking (`tests/[id]/attempt/page.tsx`)

Current: Sticky header with timer, single-question navigation, image grid, autosave.

**Changes:**
- **Timer**: Transform from red countdown to subtle pill badge in header. Normal state: muted text. Warning (<5min): gold accent. Critical (<1min): danger color. Never red by default.
- **Source Card**: When task has `has_source_text` or inherited source, render the new Source Card component above the question
- **Question text**: Larger (16px), more generous line-height (1.6)
- **Image grid**: Maintain current layout but add rounded corners and subtle shadow
- **Navigation**: Bottom bar with progress indicator (dots or thin bar), not top-heavy
- **Autosave indicator**: Tiny pill in corner, green dot when saved, not a banner
- **Submit button**: Ghost button until final question, then accent gold

## Part 3: Full Scaling (after Part 2 approved)

1. **Login/Register** — Centered card on warm background. Remove current cold gray. Add subtle product branding (product name + tagline).

2. **Tutor Dashboard** — Dense two-panel: student list (left) + analytics summary (right). Gold accent on key metrics. Student cards show progress bars with warm green.

5. **Test Taking** (all task types) — See Part 2. Matching: side-by-side columns with drag handles. Sequence: vertical numbered list with warm styling. Map tasks: image grid with numbered markers.

6. **Review** (`review/page.tsx`) — Two-column: queue (left) + detail (right). Student answer + Source Card + AI suggestion + grading form. Clear visual hierarchy between the three information sources.

7. **Analytics Dashboard** — Charts use warm palette (gold bars, green success lines). Empty state: illustrated, not just text.

8. **Content/Sync** (`content/page.tsx`) — Denser layout. Sync buttons: icon + label, not full-width. Theme tree with task counts as badges. Fix hardcoded "История" → use selectedSubject.

9. **EGE Generator** — Card with parameter inputs. Generate button: accent gold. Result: downloadable preview.

## Part 4: E2E Verification Checklist

After full redesign, verify these scenarios manually in browser:

1. **Registration**: Register new student via invitation code → consent checkbox works → redirects to dashboard
2. **Test creation**: Tutor creates test → selects positions → picks themes → test appears in list
3. **Test assignment**: Assign test to student → student sees it in their list
4. **Test taking**: Student starts test → navigates questions → autosaves → submits → sees score
5. **Image tasks**: Verify tasks with images display correctly (inherited context + own images)
6. **Source text tasks**: Verify tasks with inherited source text show Source Card
7. **Matching tasks**: Side-by-side columns render correctly, input works
8. **Essay review**: Tutor sees queue → requests AI hint → grades → submits
9. **Dashboard**: Both tutor and student dashboards load with correct data
10. **Content sync**: Sync buttons trigger correctly, status updates
11. **Responsive**: Mobile viewport (375px) — sidebar collapses, content goes full-width
12. **Accessibility**: Tab navigation works, prefers-reduced-motion disables animations

## Risk Mitigation

1. **Regression prevention**: After each screen is redesigned, verify in browser that all API calls still work (check Network tab for correct payloads)
2. **Form field names**: DO NOT rename any form field names, state variables that map to API payloads, or URL paths
3. **Image handling**: Preserve all existing image URL construction logic (`/api/v1/media/images/`)
4. **Source text display**: New feature — must gracefully handle tasks WITHOUT source text (no Source Card rendered)
5. **Docker rebuild**: After full redesign, rebuild frontend container and verify
