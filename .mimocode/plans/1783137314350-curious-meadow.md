# План: Кабинет TUTOR — 3 экрана

## Контекст

Текущее состояние:
- **"Мои тесты"** (`/tests`): список тестов с фильтрами, кнопка "Назначить" (модалка с чекбоксами), статусы назначений (цветные бейджи). НЕТ: снятие учеников, прогресс (%), просмотр ответов.
- **"Карточка теста"** (`/tests/[id]`): просмотр заданий по одному (навигация ← →), удаление заданий, список назначений сверху. НЕТ: переключение видов (одно/список), разворачивание карточек.
- **"Создать тест"** (`/tests/new`): выбор предмета, тем с деревом, кол-во на тему, тип, таймер. Counts берутся из БД (task-counts), а не с ФИПИ. Темы смешаны (История содержит Право, Обществознание, Географию из-за `CODIFIER_THEMES` в fipi_tasks.py).

**Ключевая проблема с темами**: `CODIFIER_THEMES` в `fipi_tasks.py` (строки 45-81) содержит ВСЕ темы ЕГЭ по истории, включая "10. Право", "11. Обществознание", "12. География". Это кодификатор ЕГЭ по Истории, который ВКЛЮЧАЕТ эти разделы. Но для нашего приложения (только История + Обществознание как отдельные предметы) это вызывает путаницу.

**Frontend (`tests/[id]/page.tsx`):**

- **Режим "Список":**
  ```
  №  | Кодификатор | Тип     | Текст (сокращённо)
  1  | 1.1.1       | TEST    | Кто из перечисленных учёных...
  2  | 1.2.1       | TEST    | В какой стране расположены...
  ```

## Экран 3: "Создать тест" — Реальные количества ФИПИ

### 3.1 Проблема
- `getThemeTaskCounts` считает задания ИЗ БД (таблица tasks)
- Нужно: количество заданий НА ФИПИ (в открытом банке)
- `CODIFIER_THEMES` содержит 34 темы, включая "Право", "Обществознание", "География" — это разделы кодификатора ЕГЭ по Истории

### 3.2 Решение: Новый эндпоинт для получения количеств с ФИПИ
**Backend:**
- `GET /themes/fipi-counts?subject_id={subject_id}` — количество заданий на ФИПИ для каждой темы
- Логика:
  1. Определяем project_id по предмету (История = `068A227D253BA6C04D0C832387FD0D89`)
  2. Для каждой темы из CODIFIER_THEMES делаем POST к `questions.php` с `pagesize=1`
  3. Парсим HTML, считаем количество `div.qblock` (общее) + разбивку по типам
  4. Кэшируем результат на 1 час (Redis или in-memory)

**Проблема**: Это ~34 HTTP-запроса при выборе предмета. Нужно:
- Кэширование (Redis, TTL 1 час)
- Или: предзагрузка при старте приложения
- Или: фоновая задача Celery, которая обновляет counts

**Рекомендация для MVP**: Кэширование в Redis с TTL 1 час. При первом запросе — загрузка всех counts, при последующих — из кэша.

### 3.3 Разделение тем по предметам (решение: отдельный mapping)
**Проблема**: Темы "Право" (10.x), "Обществознание" (11.x), "География" (12.x) относятся к ЕГЭ по Истории, но в нашем приложения — это отдельные предметы.

**Решение (выбрано пользователем)**: Создать отдельный mapping тем для Обществознания.

**Кодификатор ЕГЭ по Обществознанию** (отдельный от Истории):

**FIPI Project ID для Обществознания**: `068A227D253BA6C04D0C832387FD0D89` (тот же, что и История — ЕГЭ сдаётся вместе)

**Изменения в `fipi_tasks.py`**:
- `CODIFIER_THEMES` → переименовать в `CODIFIER_THEMES_HISTORY` (темы 1-9)
- Добавить `CODIFIER_THEMES_SOCIAL` (отдельный mapping)
