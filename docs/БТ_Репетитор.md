# Бизнес-требования (BRD): Комбинатор тестов ФИПИ + Аналитика прогресса (MVP)

## 1. Проблема и Контекст

**Проблема:** Репетиторы тратят часы на рутинную работу: ручной поиск заданий в банке ФИПИ, сборку вариантов, проверку типовых частей и ведение аналитики прогресса в Excel/тетрадях. Это снижает эффективность подготовки и не дает ученикам/родителям прозрачной картины успеваемости.

**Контекст:** Первый пользователь — один репетитор по истории/обществознанию. Продукт должен с первого дня поддерживать ролевую модель и структуру данных для будущего масштабирования (мультитенантность), но в MVP реализуется в режиме «закрытого прототипа» (один репетитор).

## 2. Пользовательские истории (User Stories) и Критерии приёмки

### 👤 Роль: Репетитор

**US-1: Сборка теста из банка ФИПИ**

> *Как репетитор, я хочу собирать тесты, выбирая темы кодификатора, типы заданий и лимит времени, чтобы быстро формировать индивидуальные тренировочные варианты для учеников.*

* **Критерии приёмки:**

* Система позволяет выбрать один или несколько предметов (История, Обществознание).

* Система отображает дерево тем/подтем согласно кодификатору ФИПИ.

* Можно задать количество заданий на тему, тип (тестовая часть / развернутая часть).

* **Можно установить общий таймер на прохождение теста (в минутах). Таймер опционален — можно создать тест без ограничения по времени.**

* Сгенерированный тест сохраняется в списке тестов репетитора.

**US-2: Назначение теста ученику**

> *Как репетитор, я хочу назначать созданный тест конкретному ученику (или группе учеников), чтобы они могли приступить к тренировке.*

* **Критерии приёмки:**

* Из карточки теста можно выбрать ученика из списка привязанных к репетитору.

* Ученику доступна ссылка для прохождения или уведомление в Личном кабинете.

**US-3: Проверка развернутых ответов**

> *Как репетитор, я хочу проверять развернутые ответы ученика с помощью чек-листа критериев ФИПИ и AI-подсказок, чтобы тратить на проверку минимум времени.*

* **Критерии приёмки:**

* Тестовая часть проверяется автоматически сразу после сдачи (сверка с ключами ФИПИ).

* Развернутые ответы поступают в очередь «На проверке».

* В UI проверки отображается: текст задания, ответ ученика, официальные критерии ФИПИ.

* AI (GigaChat) выдает текстовую подсказку-разбор соответствия ответа критериям (но не ставит итоговый балл).

* Репетитор вручную проставляет баллы по каждому критерию и итоговый балл за задание.

**US-4: Дашборд прогресса и аналитика**

> *Как репетитор, я хочу видеть единую панель по всем своим ученикам, чтобы отслеживать динамику и выявлять пробелы в знаниях.*

* **Критерии приёмки:**

* Доступен список всех учеников репетитора с общим % прохождения тестов.

* В профиле ученика отображается график динамики баллов по датам.

* Выводится разбивка по темам кодификатора (сильные/слабые темы на основе % правильных ответов).

* Учитываются все попытки прохождения (ретесты). Показывается прогресс (было -> стало).

---

### 🧑‍🎓 Роль: Ученик

**US-5: Регистрация и привязка к репетитору**

> *Как ученик, я хочу зарегистрироваться и подтвердить согласие на обработку данных (с учетом 152-ФЗ), чтобы получить доступ к тестам.*

* **Критерии приёмки:**

* Форма регистрации требует Email, Пароль, ФИО, Дату рождения.

* Обязательный чекбокс «Согласен на обработку ПДн» (ссылка на Политику).

* Возможность ввести «код приглашения» от репетитора (или репетитор сам создает профиль ученика и отправляет ссылку для активации).

**US-6: Прохождение теста (UI)**

> *Как ученик, я хочу удобно проходить тест с сохранением черновиков ответов, чтобы не потерять работу при сбое.*

* **Критерии приёмки:**

* Доступен таймер обратного отсчета. При 00:00 тест сдается автоматически.

* Можно переходить между заданиями, менять ответы.

* Ответы автоматически сохраняются на сервер при каждом изменении.

* После отправки тестовая часть моментально проверяется, ученик видит балл за неё. Развернутые задания помечаются статусом «Ожидает проверки репетитором».

**US-7: Ретесты (Повторные прохождения)**

> *Как ученик, я хочу иметь возможность пересдавать темы неограниченное количество раз, чтобы закрепить материал.*

* **Критерии приёмки:**

* Система не блокирует повторное прохождение теста/темы.

* Каждая новая попытка сохраняется отдельно и учитывается в общей аналитике.

---

> **Роль «Родитель» исключена из MVP управленческим решением.** Роль и связанные с ней сценарии (просмотр упрощённого дашборда ребёнка) переносятся в бэклог пост-MVP. Ниже — сохранённая формулировка будущей истории, для справки при планировании следующей итерации:
>
> *US-8 (пост-MVP): Как родитель, я хочу видеть прогресс своего ребенка без необходимости спрашивать репетитора, чтобы понимать эффективность занятий. Родитель видит только упрощённый дашборд ребёнка (динамика баллов, % успеха по темам), не видит тексты заданий, развёрнутые ответы и данные других учеников.*

## 3. Критерии успеха (Измеримые)

1. **Скорость сборки теста:** Репетитор может собрать тренировочный вариант на 20 заданий из банка ФИПИ менее чем за **3 минуты** (без ручного копирования текстов).

2. **Сокращение времени на проверку:** Использование AI-подсказок и UI с чек-листами сокращает время проверки одного развернутого ответа до **30-45 секунд**.

3. **Adoption (Вовлеченность):** Ученик совершает минимум 2 повторные попытки (ретеста) по слабым темам за учебный месяц.

## 4. Скоуп (MoSCoW)

### 🟢 MUST HAVE (Обязательно для MVP)

* Парсер открытого банка ФИПИ (синхронизация базы заданий по темам/кодификатору для Истории и Обществознания).

* Конструктор тестов (выбор тем, типов, таймера).

* Автоматическая проверка тестовой части (сверка с ключами).

* UI для репетитора по ручной проверке развернутых ответов с AI-комментарием.

* ЛК Ученика (прохождение, сохранение ответов, просмотр результатов).

* Базовая аналитика (динамика баллов, % правильных ответов по темам).

* Безлимитные ретесты.

* Роли: Репетитор, Ученик.

* Серверное хранение результатов.

### 🟡 SHOULD HAVE (Желательно, если позволит время)

* Экспорт результатов ученика в PDF/Excel для репетитора.

* Email/Telegram уведомления репетитору о сдаче теста учеником.

* Подсветка «любимых ошибок» ученика в рамках одной темы.

### 🔴 WON'T HAVE (Исключено из MVP)

* Мультитенантность (маркетплейс репетиторов, изоляция данных между разными независимыми репетиторами).

* Предметы вне кодификатора Истории и Обществознания.

* Интеграция оплат, подписок, биллинга.

* Полная автоматическая оценка развернутых ответов AI (AI только помогает).

* Нативные мобильные приложения (iOS/Android).

* Учет веса/сложности заданий в базовой метрике (считаем просто % правильных ответов).

## 5. Ограничения и Приватность (152-ФЗ)

* **ПДн несовершеннолетних и Возрастные ограничения:** Так как ученики часто младше 18 лет, система требует явного согласия на обработку персональных данных. **Однако, в рамках MVP (закрытого прототипа) жесткая верификация возраста и блокировка регистрации для пользователей до 14 лет не реализуются.** Риски митигируются юридически корректными текстами оферты и чекбоксами при регистрации.

* **Минимизация данных:** Мы не храним паспортные данные или СНИЛС. Только Email, ФИО, Дата рождения, Привязки (Репетитор-Ученик) и Результаты тестов. Связь `student_parent` в MVP не заполняется — таблица зарезервирована в БД как задел на будущее (см. документ Архитектуры).

* **Хранение:** Все данные хранятся на сервере (согласно требованию локализации 152-ФЗ — серверы на территории РФ).

* **MVP (закрытый прототип):**

* Реализуем только получение согласия на обработку ПДн при регистрации (чекбокс + ссылка на Политику).

* **Функционал "Удалить аккаунт" / "Отозвать согласие" не реализуем в MVP.**

* Удаление данных осуществляется вручную администратором по запросу пользователя.

* **Пост-MVP (при масштабировании):**

* Добавим self-service удаление аккаунта и отзыв согласия в Личном кабинете.

* Реализуем автоматическое удаление данных через 30 дней после отзыва согласия.
