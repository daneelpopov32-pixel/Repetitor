## 4. Проблемы, баги и недоработки

### Критические

| Проблема | Описание |
|----------|----------|
| **FIPI блокирует скачивание изображений** | При повторном запросе FIPI возвращает одинаковый HTML для всех тем. ShowPictureQ() не извлекаются. 78 файлов скачаны ранее (только тема 4). Задания Типов 8-12, 15-16 требуют картинки |
| **274 задания без классификации** | Short_answer без структурных маркеров (7.8% базы) |

### Исправленные в этой сессии

| Проблема | Решение |
|----------|---------|
| `exam_position` не приходил с API | Добавлен в `get_test` и `get_attempt` |
| `_FIGURE_PATTERNS` ловил варианты ответов | Проверяю только описание задания (`_get_task_description`) |
| `_MAP_PATTERNS` слишком широкий | Заменён на `_TYPE9_12_MAP_INTRO` — только "на схеме/карте" |
| `_TABLE_PATTERNS` ловил пропуски в предложениях | Убраны generic-паттерны |
| short_answer → дефолт позиция 3 | Убран дефолт, честный None |
| Essay без маркера → None | Добавлен дефолт → позиция 14 |
| Тип 6 не матчился (subtype=essay) | Добавлен в essay-секцию |
| Тип 12 и 16 пересекались | Разделены: 12="суждения", 16="здания/сооружения" |
| `docs/типизация.md` Тип 12 неверный | Исправлено |

## 5. Допущения

1. **Типы 9-12 без изображений** — классифицируются по текстовому маркеру, хотя по спецификации требуют физическое изображение. Компромисс из-за блокировки FIPI.

2. **274 null-задания** — принято как допустимое (7.8%). Альтернатива — добавить дефолт, но это приведёт к неверной классификации.

3. **Тип 14 как дефолт для essay** — не все essay являются "извлечением информации из источника". Но альтернатива (None) хуже.

4. **FIPI может изменить API** — парсер хрупкий, зависит от структуры HTML. Любое изменение на стороне ФИПИ сломает синхронизацию.
